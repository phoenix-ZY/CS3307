支持的库：
pandas re enchant nltk tqdm

其中，使用pip 安装nltk后，初次运行时需要运行nltk.download()(已写在代码中），然后下载所有包，之后
运行时可以注释掉此行（不注释不影响运行，只是降低运行速度）
tqdm提供三个模块的运行进度和预计运行时间
有一定概率报错 表示你的numpy版本太低或者太高，我记得是需要1.19.0-1.25.0之间，我的是1.23.0

使用：
main函数中有两个实例，分别用来处理csv文件和test用。
具体使用方式如下：
由于本人封装技术有限，需要创建类目标：obj = preprocess.preprocess()
preprocess.preprocess.get_text(obj,url,drop=Ture,samplenum=10000,save=False)
drop表示是否需要舍去一部分数据，samplenum表示需要留下的数据量，save表示是否需要将处理结果
保存到本地（如果一次性处理全部测试数据，强烈推荐设置为Ture）
testandtry（obj，text）提供一个目标量较小的测试模块，会输出处理结果。参数为list。

问题：
拼写错误：不需要处理，首先需要根据上下文，处理量大，并且有一定的处理错误率。
通俗英语拼写习惯：如缩写，或者情绪激动下打出来的loooovvveeeee sooooo cuteeeeee这种，暂时不
处理，当然会增加向量化之后的无意义数据量，但是对于大部分缩写，停用词列表会过滤一部分，剩下我
使用字典和正则表达删除，如果是人称，代词之类，不影响情感分析。至于对于比较重要的形容词和动词
产生的typo，忽视一部分难以处理的，剩下我会想想办法，，
